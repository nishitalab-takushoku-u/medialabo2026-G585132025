「WEBコンテンツ作成」の成果

本テーマについて

このテーマでは、HTML,CSS,JavaScriptの基本を学び、これらを組み合わせてWebページを構成する方法を学びます。目標としてはWebページの構造を記述する基本的な方法や,Webページの見て目の設定の基本的な方法などがわかるなどが挙げられます。

プロジェクト課題について

この科目を通して学んだHTML,CSS,JavaScriptをもとにプロジェクト課題に取り組みました。個人的に、HTMLを作ってCSSを作ったところまでは順調に進んでたと感じていましたが、HTMLで書いたものをJavaScriptに移し替える作業にすごく苦戦しました。JavaScriptに入ってから基本的なコードがとても多くなって、似てるものも多く使い分けがうまくできていなかったので苦戦しました。

プロジェクト課題で作成したページ: project.html

数当てゲーム

数当てゲームは、コンピュータに10以下の乱数で答えを決めてもらい、ユーザがその答えを最大3回以内に予想するゲームです。ユーザの入力が外れた場合は、ヒントとして入力した値が答えよりも答えが大きいか小さいかが表示されます。3回以内に答えの数字を当てたらユーザの勝ち、もしそうでなければユーザの負けとなります。ページにはテキストボックスとボタンが設置されています。ユーザは1から10の間の整数と書かれた右にあるテキストボックスに予想を入力して回答というボタンを押すと正解か否かの判定が表示されます。このゲームを作るにおいて、特に初めて学ぶことや初めて見るプログラム自体は多くなく、今までの復習のような形になりました。

数当てゲームのページ: kazuate.html

各授業の学習事項

第1回 WWW と HTML

第1回目の授業で学んだことは、WWWとHTMLですがHTMLの少し詳しく学びました。具体的には見出しの作り方やリストの作り方、画像の添付の仕方や、ハイパーリンクの作り方などです。自分的にはハイパーリンクに少し苦戦しました。

演習の成果: 演習1-4: HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

第2回目で学んだことはCSSでWEBページをどのように装飾するかということです。この回では新たにCSSというものが出てきました。具体的には文字の色やフォントを変更できたり、特定の要素を指定したりなどをできます。

演習の成果: 演習2-3: ウェブブラウザ説明ページの装飾

第3回 JavaScript の基本

第3回目の授業で学んだことは、JavaScriptのすごい基本的なところを学びました。JavaScriptでできることや、JavaとJavaScriptの違いや、簡単な構文の使い方、デバックの仕方などです。癖でJavaの構文を使わないように気をつけたいです。

演習の成果: 演習3-6: オブジェクトの参照

第4回 DOM と Javascript

第4回目ではフォームの要素、イベントの処理、DOM(Document Object Model)について学びました。この回では、要素を検索だったり、要素のテキストを参照するなどといった構文を教わりました。前回からJavaScriptを使うようになり、使用する構文が増えましたが、今回から特に増えてきて難しく感じてきました。

演習の成果: 演習4-10: 足し算ページ

第5回 イベント処理

第5回の授業では、新たに5つの構文を学びました。1つ目は要素の新規作成、2つ目は要素1の内部に要素2を配置するというもの。3つ目は内部ではなく直後に配置するもの。4つ目は、要素の属性を設定、5つ目は要素を削除するものです。

演習の成果: 演習5-7: DOMプログラム(2)

工夫点、苦心点、強調点、感想

工夫点

この授業を通して、工夫点として自分なりに見やすいページを作ろうと常に思いながらプログラムを打ってきました。

苦心点

苦心点はいくつもあります。一つ目は単純に習うプログラムが多くて覚えることもできないで講義資料と常に睨めっこだったということです。2つ目は、自分なりに見やすいページを作ろうと思ってプログラムを打っていても平凡すぎるなんの特徴もないページになってしまったことです。3つ目は自分の問題点でもあるんですが、似たようなプログラムとして紹介されたり習ったものをちゃんと講義資料を読まないで使ってエラーを出して頭を抱える時間が長かったということです。リンクを貼る時、そのリンクが自分のフォルダ内なのかどうかというところを読まずに使って時間を食っていました。4つ目として、JavaScriptのプログラムとJavaのプログラムが似ているところもあるけど全然違うというところです。if,else文の構成や計算の仕方などはほぼ同じでしたが、文字の表示の仕方や、定義の仕方などが全然違くて苦戦しました。

感想

私がWebコンテンツの作成の基礎を学んで感じたことは、いつも見ているWebページにはとてもたくさんの時間がかけられていて、制作者はページを見る側のことを考えてデザインなどを決めているのだなということです。苦心点でも書いた通り、プログラム自体が多くて覚えることが難しく、使いこなすのが難しかったことはもちろんなのですが、それよりも自分で作ったページがいつも授業などで使わせていただいているWebページや、趣味や勉強などで使っているページのデザインや使いやすさのようには全然できなかったというところからそう思いました。Webページひとつ作るにもHTML,CSS,JavaScriptの3つから構成されていてそれぞれに結構な量のプログラムを打ち込んでデバックして打ち込んだプログラムが正常に作動しているかどうかを見て、ページもうまく動いているかを確かめるなどの作業もしないといけなくてとても大変だと感じました。この授業ではfirefoxのWeb開発ツールのコンソールというとても便利な機能を使って学びながらページを作りましたが、これがないといまだに出来る気がしないのが現状です。

最後に

これから趣味などでWebページなどを作ろうと軽い考えでこの授業を受講してみましたが、結構辛いことがわかったのでもっと勉強して見る人にとって優しいページを作れるようになりたいと思います。
